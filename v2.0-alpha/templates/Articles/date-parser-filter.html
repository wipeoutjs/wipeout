A two way date formatter

The following example is of a date textbox where the user can choose the format of the date they enter. It uses both the upward and downward filter parts.

// simple function to convert "5" to "05" etc... function addZeroes (value, requiredLength) { value = value.toString(); while (value.length < requiredLength) value = "0" + value; return value; } // add filters to wo.filters wo.filters.dateParser = { // This filter parses from strings to dates downward: function (date, format) { if (format === "dd/mm/yyyy") { return addZeroes(date.getDate(), 2) + "/" + addZeroes(date.getMonth() + 1, 2) + "/" + addZeroes(date.getYear() + 1900, 4); } else if (format === "mm/dd/yyyy") { return addZeroes(date.getMonth() + 1, 2) + "/" + addZeroes(date.getDate(), 2) + "/" + addZeroes(date.getYear() + 1900, 4); } else { return date; } }, // This filter parses from dates to strings upward: function (dateString, format) { if (format === "dd/mm/yyyy") { var date = new Date(); date.setYear(dateString.substr(6)); date.setMonth(parseInt(dateString.substr(3, 2)) - 1); date.setDate(dateString.substr(0, 2)); return date; } else if (format === "mm/dd/yyyy") { var date = new Date(); date.setYear(dateString.substr(6)); date.setMonth(parseInt(dateString.substr(0, 2)) - 1); date.setDate(dateString.substr(3, 2)); return date; } else { return date; } } }; <wo.content-control model="new Date()" format--s="dd/mm/yyyy"> <set-template> Date format: <input type="radio" value="dd/mm/yyyy" wo-value="$this.format" name="format" /> dd/mm/yyyy <input type="radio" value="mm/dd/yyyy" wo-value="$this.format" name="format" /> mm/dd/yyyy <br /> <br /> Enter date in correct format: <input type="text" wo-value="$model, $this.format => dateParser" wo-on-event="blur" /> <br /> Actual value: <b wo-content="$model"></b> </set-template> </wo.content-control>

In the above example, the filter is defined as follows:

  1. The first argument is the primary argument. When the value in the textbox changes, it is the value which will be set.
  2. All subsequent arguments are simple filter values.
  3. The argument after the => is the filter.
