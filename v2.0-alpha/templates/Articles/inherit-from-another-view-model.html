Inherit from another viewModel

At it's core, wipeout uses an Object Oriented Programming (OOP) model to build types of viewModels. This means that your viewModels must inherit from another viewModel type in order to use wipeout functionality. The default and most basic viewModel to inherit from is however you can change this with the second argument of wo.viewModel

// inherit from contentControl wo.viewModel("myNamespace.myViewModel", wo.contentControl) .build();

Inherit from a custom viewModel class

You can also inherit from a viewModel class which you have created

var myParentViewModel = wo.viewModel("myNamespace.myParentViewModel") .build(); myParentViewModel.getClassName = function () { return "myParentViewModel"; }; // inherit from myNamespace.myParentViewModel var myChildViewModel = wo.viewModel("myNamespace.myChildViewModel", myNamespace.myParentViewModel) .build(); myChildViewModel.getClassName = function () { // you can use the "_super" function to call the method of a parent class var parent = this._super(); return "myChildViewModel, inherits from " + parent; };

There is a caveat to using the _super(...) function. Internally _super jumps onto the function call stack, and this behavior is not allowed if writing javascript in strict mode.
