One way bindings

The default binding is one way. This means that the object where the propery is set will recieve updates when the property changes, however, if the object where the property is set changes the value itself, the original owner of the property will not be notified. In most cases properties will bound one way by default, but occasionally (for instance if a is set for a property) you may need to explicitly set the binding to one way using the --ow .

<wo.content-control model="busybody.makeObservable({ value: 'Hello wipeout!' })"> <set-template> <wo.content-control model--ow="$this.model.value"> <set-template> One way bindings: <span wo-content="$model"></span> </set-template> </wo.content-control> </set-template> </wo.content-control> <wo.view template-id="'bind-one-way-script-1'" />
