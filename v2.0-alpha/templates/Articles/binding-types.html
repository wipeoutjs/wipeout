Binding types

There are many ways of binding properties. By default properties are bound one way, but properties can also be bound

One way binding (default)

<wo.content-control model="busybody.makeObservable({ innerValue: 'Hello wipeout!' })"> <set-template> Parent: <input wo-on-event="keyup" wo-value="$model.innerValue" /><br /> <wo.content-control model--ow="$this.model.innerValue"> <set-template> Child: <input wo-on-event="keyup" wo-value="$model" /> </set-template> </wo.content-control> </set-template> </wo.content-control>

Notice how changes to the parent affect the child, but changes to the child do not affect the parent

<wo.view template-id="'binding-types-ow'"></wo.view>

Two way binding

<wo.content-control model="busybody.makeObservable({ innerValue: 'Hello wipeout!' })"> <set-template> Parent: <input wo-on-event="keyup" wo-value="$model.innerValue" /><br /> <wo.content-control model--tw="$this.model.innerValue"> <set-template> Child: <input wo-on-event="keyup" wo-value="$model" /> </set-template> </wo.content-control> </set-template> </wo.content-control>

Notice how changes to the parent and child affect eachother.

<wo.view template-id="'binding-types-tw'"></wo.view>

One way to source binding

<wo.content-control model="busybody.makeObservable({ innerValue: 'Hello wipeout!' })"> <set-template> Parent: <input wo-on-event="keyup" wo-value="$model.innerValue" /><br /> <wo.content-control model--owts="$this.model.innerValue"> <set-template> Child: <input wo-on-event="keyup" wo-value="$model" /> </set-template> </wo.content-control> </set-template> </wo.content-control>

Notice how changes to the child affect the parent, but changes to the parent do not affect the child

Notice how changes to parent and child do not affect eachother
