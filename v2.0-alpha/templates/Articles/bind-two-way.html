Two way bindings

By default, all bindings are one way. This means that the object where the propery is set will recieve updates when the property changes, however, if the object where the property is set changes the value itself, the original owner of the property will not be notified. This can be remedied by using a two way binding. Two way bindings use the --tw .

<wo.content-control model="busybody.makeObservable({ value: 'Hello wipeout!' })"> <set-template> <wo.content-control model--tw="$model.value"> <set-template> Two way bindings: <span wo-content="$model"></span> </set-template> </wo.content-control> </set-template> </wo.content-control> <wo.view template-id="'bind-two-way-script-1'"></wo.view>

Now if the child content control modifies it's model, the resulting change will reflect in the parent's model.value. Notice how the model is created using busybody.makeObservable in this case. Two way bindings can only be defined between two observable objects. If this is not the case, wipeout will make a best attempt to bind the two properties.
