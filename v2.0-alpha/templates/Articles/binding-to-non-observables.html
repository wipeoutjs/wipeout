Binding to non observables

When something is bound within a template, part of the process is to watch the variable for change, so that the DOM, or other view models can be updated accordingly. Plain old javascript objects cannot be watched, they must first be converted to observables with busybody.makeObservable.

Strategies

There are three binding strategies you can use

1. Create Observables (default)

This strategy will bind to anything. Given the path: $model.item1.item2, where neither $model nor item1 are observables, wipeout will convert these values to observables before binding so that any changes to $model.item1 or item1.item2 can be observed. Once the binding is destroyed or an item is no longer present in the path, the item will then be converted back to a non observable.

There is overhead in creating observables, and if you are building a large SPA with many long binding paths, this strategy might not be optimal.

2. Bind non Observables

This strategy will bind to anything that it can. Given the path: $model.item1.item2, where item1 is an observable, but $model is not, wipeout will do it's best to bind. This means that if item1.item2 changes, wipeout will update the DOM, however, if $model.item1 changes, wipeout will not.

3. Only Bind Observables

This strategy will only bind until it reaches a break in the observable chain. Given the path: $model.item1.item2, where item1 is an observable, but $model is not wipeout will not bind, as the first link in the chain ($model) is not observable.
If $model were to be observable but item1 were not, wipeout act the same as if Bind non Observabls were used, altering the DOM if $model.item1 changes, but not if item1.item2 changes.
