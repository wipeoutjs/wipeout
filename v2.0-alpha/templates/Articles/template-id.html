Templating a wo.view

In MVVM application design the viewModel layer is always necessary, but not always useful. This is generally the case when there is no view logic (dom manipulation) to be done and in wipeout, is generally the case when you want to template a model or sub model without the overhead of creating a new viewModel class.
If this is the case you can template a wo.view directly.

<wo.content-control model="{title: 'Hello wipeout', subModel: { ... }}"> <set-template> <wo.viewModel="$parent.model.subModel" template-id="'myTemplateId'" /> </set-template> </wo.content-control>

Notice the single quotes surrounding the template id property. Anything defined in a property on a viewModel or a "data-bind" property on a html element is compiled into javascript, so the single quotes denote a string.

Using this method, you can re-template any viewModel at any time by setting it's template id.
